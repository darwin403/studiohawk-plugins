# Ranking Analysis

Use this when a ranking in the press release looks off, or the user asks "why is **Item A** ranked
above/below **Item B**". Answer using **only** the methodology document and the data file. If those
two don't contain what's needed to answer, say so directly instead of guessing.

## Show your working

1. **List every factor** the methodology uses to build the ranking, each with its exact weighting.
2. **Compare the two items** on each factor, pulling the raw values straight from the data file
   (cite the tab and cells).
3. **Apply the weightings exactly** as the methodology states them — do not invent, round, or
   adjust weightings. Show the per-factor contribution and the resulting totals.
4. **Explain in plain English** which factors drove the result, and by how much (e.g. "A leads
   almost entirely on <factor>, which carries <weight>; B is ahead on <factor> but it's weighted
   lower").
5. **Flag relevant limitations** — any methodology caveat that bears on this specific comparison,
   so the story doesn't claim more than the analysis supports.

## Guardrails

- If the computed order disagrees with the ranking shown in the data file or the press release,
  report the discrepancy and show both — don't paper over it.
- If the methodology is silent on something the comparison needs (a tie-breaker, a missing factor),
  state that gap explicitly rather than filling it with an assumption.
- Keep every number traceable to a cell and every weighting traceable to a methodology line.
